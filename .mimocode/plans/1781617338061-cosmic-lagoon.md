## Critical Files for Security Audit

### Electron Security
- `electron/main.ts:378`: DEEPSEEK_API_KEY passed to spawn env
- `electron/main.ts:270`: killStaleProcess() uses execSync for process killing
- `electron/main.ts:1054-1058`: webPreferences settings (contextIsolation, nodeIntegration)

### API & Network Security
- `electron/ocr-client.ts:12-14`: HTTP-only connection to localhost:5002
- `backend/ocr_server.py:62-66`: DeepSeek API configuration and endpoints
- `electron/main.ts:621-637`: Tencent API request with GBK decoding (hardcoded)
- `electron/main.ts:744`: Sina API request (hardcoded)
- `backend/services/market_data/stock_api.py`: Multiple hardcoded providers

### Custom API Feature - New Requirement
- Users should be able to configure custom stock data APIs
- Settings UI in top-right corner
- Replace hardcoded Tencent/Sina APIs with configurable endpoints
- Support multiple API formats (custom JSON structure)

### Data Persistence & Privacy
- `src/App.tsx:307-320`: Auto-save to SQLite history
- `electron/main.ts:920-951`: File export to local/exports/
- `backend/ocr_server.py:830-945`: History CRUD operations

### Logging & Information Disclosure
- `electron/main.ts:73-91`: Log file writing to user data directory
- `backend/ocr_server.py:79-105`: Request logging with path and parameters

## Security Checklist to Verify

### High Priority
- [ ] API keys not exposed in logs or UI
- [ ] No shell injection in execSync calls
- [ ] Input validation on all IPC handlers
- [ ] Secure file permissions on user data
- [ ] Custom API configuration validation

### Medium Priority
- [ ] HTTPS for local API (or ensure localhost-only)
- [ ] Sanitize logs (remove sensitive paths)
- [ ] Validate file paths before operations
- [ ] Handle errors gracefully without information leakage

### Low Priority
- [ ] Cleanup temporary files
- [ ] Secure defaults for file permissions
- [ ] Rate limiting on API endpoints

## Verification Steps

### Manual Testing
1. Run application and check logs for API key exposure
2. Test file export functionality
3. Verify history CRUD operations work correctly
4. Check Electron security settings in dev tools

### Automated Testing
1. Run existing tests: `npm run test`

3. Check linting: `npm run lint` (if available)

### GitHub Upload Preparation
1. Ensure no secrets in code or config
2. Add .gitignore for sensitive files
3. Document security measures in README

## Custom API Configuration Feature

### User Request: Custom Stock Data APIs
**Requirement:** Allow users to configure custom API endpoints instead of hardcoded Tencent/Sina APIs

**Implementation Plan:**

#### 1. Settings UI (Top-Right Corner)
- Add gear icon button in main UI header
- Open settings panel with tabs:
  - **"API Configuration"** - Custom stock data APIs
  - **"Privacy Settings"** - Data retention, AI processing consent
  - **"About"** - App information

#### 2. API Configuration Structure
```typescript
interface StockApiConfig {
  realtimeQuote: {
    url: string;  // Template: "https://api.example.com/quote?code={code}"
    method: 'GET' | 'POST';
    headers?: Record<string, string>;
    responseParser: 'json' | 'text';
  };
  klineData: {
    url: string;  // Template: "https://api.example.com/kline?code={code}&period={period}"
    method: 'GET' | 'POST';
  };
  stockSearch: {
    url: string;  // Template: "https://api.example.com/search?keyword={keyword}"
  };
}
```

#### 3. Files to Modify
- `src/App.tsx` - Add settings button and panel
- `src/components/SettingsPanel.tsx` (NEW) - Settings dialog component
- `src/components/ApiConfigTab.tsx` (NEW) - API configuration tab
- `src/services/market-service.ts` - Use configurable API endpoints
- `electron/main.ts` - Store/retrieve API config in user data
- `electron/preload.ts` - Expose config API to renderer
- `backend/ocr_server.py` - Accept custom API config from frontend

#### 4. Default Configuration (Built-in Fallback)
```typescript
const defaultApis = {
  tencent: {
    name: '腾讯行情',
    realtimeQuote: { url: 'http://qt.gtimg.cn/q={code}', method: 'GET' },
    klineData: { url: 'http://money.finance.sina.com.cn/...', method: 'GET' }
  },
  sina: {
    name: '新浪行情',
    realtimeQuote: { url: 'http://hq.sinajs.cn/list={code}', method: 'GET' },
    klineData: { url: '...', method: 'GET' }
  }
};
```

#### 5. User Experience Flow
1. User clicks gear icon in top-right corner
2. Settings panel opens with "API Configuration" tab
3. User can:
   - Select from built-in providers (Tencent, Sina)
   - Add custom API endpoint with URL template
   - Configure request headers (for authentication)
   - Test connection with "Test API" button
4. Configuration saved to user data directory
5. App uses custom API if configured, fallback to defaults

#### 6. Security Considerations for Custom APIs
- Validate URL format (must be http/https only)
- Sanitize headers to prevent injection attacks
- Rate limit API requests
- Log API responses (without sensitive data)
- Timeout handling for custom APIs
